各運用マニュアルを生成させるためのプロンプトです。

各プロンプトを丸ごとコピーして生成AIに貼り付けてください。
画像・スクリーンショットが必要な箇所は、実際の画面キャプチャに差し替えてください。

# プロンプト①：初期導入マニュアル

```
あなたはSaaS製品の運用マニュアル作成の専門家です。
以下のシステム情報をもとに、IT知識のない飲食店・美容室・整体院などの店舗オーナーが
一人でシステムを導入できる「初期導入マニュアル」を作成してください。

## システム概要

「LINE連動型予約システム」は、LINEアプリから予約できるシステムです。
仕組みは以下の通りです：
- お客様はLINEの公式アカウントを友だち追加し、LIFFアプリ（LINE内ブラウザで開くWeb画面）から予約します
- 予約データはGoogleスプレッドシートに保存されます
- 予約が入るとGoogleカレンダーに自動でイベントが作成されます
- 店舗スタッフはGoogleアカウントでブラウザから予約管理画面にアクセスします
- システムはGoogleのクラウドで動作します（サーバー契約不要）

## 導入手順（正確に記載してください）

### STEP 0：事前準備（導入前に必ず実施）
導入を始める前に、使用するGoogleアカウントでApps Script APIを有効化する必要があります。
これを行わないとシステムの自動構築が失敗します。

1. 以下のURLをブラウザで開く：https://script.google.com/home/usersettings
2. 「Google Apps Script API」のトグルをクリックして**オン**にする
3. 画面が切り替わり有効になったことを確認する

⚠️ この設定は導入に使用するGoogleアカウントごとに1回だけ行う必要があります。
⚠️ 有効化直後は数分待ってから次のSTEPに進んでください。

### STEP 1：導入サイトにアクセス
- 導入URL：https://reservation-onboarding.reservation-onboarding.workers.dev/
- 「Googleアカウントで導入開始」ボタンをクリック
- Googleのログイン画面が開くのでSTEP 0で設定したGoogleアカウントでログイン
- Googleのアクセス許可確認画面が出たら「続行」をクリック
- 自動でシステムが構築され、完了画面にURLが表示される（約30秒）

### STEP 2：表示されたURLを控える
完了画面に以下の2つのURLが表示されます：
- 「GAS WebアプリURL」（例：https://script.google.com/macros/s/AKfycb.../exec）
- 「スプレッドシートURL」（Googleスプレッドシートへのリンク）
⚠️ 必ずメモまたはブックマークしてください

### STEP 3：GASの初回認証を行う
1. 完了画面に表示された「GAS WebアプリURL」をクリック（または貼り付けてアクセス）
2. 「このアプリがGoogleアカウントへのアクセスを求めています」という画面が表示される
3. 「許可」をクリック
4. 設定画面（予約システム設定画面）が表示されれば認証完了
⚠️ この操作はSTEP 0でApps Script APIを有効化したGoogleアカウントで行ってください

### STEP 4：設定画面（GAS WebアプリURL）から初期設定を行う
STEPで表示された設定画面から以下の値を入力します：
（スプレッドシートを直接編集する必要はありません）

| 設定項目 | 入力内容 | 具体例 |
|---|---|---|
| shopName | 店舗名 | サンプル美容室 |
| businessHours | 営業時間（JSON形式） | {"start":"10:00","end":"19:00"} |
| holidays | 定休日（0=日曜〜6=土曜） | [0,1]（日・月が定休の場合） |
| bookingTimeUnit | 予約枠の単位（分） | 15 |
| maxConcurrentBookings | 同時予約上限数 | 2 |
| bookingLookaheadDays | 何日先まで予約できるか | 60 |
| serviceMenus | メニューと所要時間（JSON形式） | [{"name":"カット","duration":60},{"name":"カラー","duration":90}] |
| adminEmail | 管理者のGmailアドレス | yourname@gmail.com |
| isStaffFeatureEnabled | 担当者機能（TRUE/FALSE） | TRUE |
| staffs | 担当者一覧（JSON形式） | [{"name":"田中","email":"tanaka@gmail.com"}] |

### STEP 5：Googleカレンダーの設定
1. Google カレンダー（https://calendar.google.com）を開く
2. 予約専用の新しいカレンダーを作成（例：「予約カレンダー」）
3. カレンダーの設定画面から「カレンダーのID」をコピー（例：xxxxx@group.calendar.google.com）
4. スプレッドシートのConfigシートの「reservationCalendarId」に貼り付け

### STEP 6：LINE公式アカウントのWebhook設定
1. LINE Developersコンソール（https://developers.line.biz）にログイン
2. 使用するチャンネルを開く
3. 「Messaging API設定」タブを開く
4. 「Webhook URL」に以下を入力：
   STEP 2で控えたGAS WebアプリURL（そのまま貼り付け）
5. 「Webhook URLを更新」→「検証」ボタンで成功を確認

### STEP 7：設定画面にLINEのトークンを設定
LINE Developersコンソールから以下をコピーして設定画面（GAS WebアプリURL）に入力：
- LINEチャンネルアクセストークン
- LINEチャンネルシークレット
- LIFF ID

### STEP 8：動作確認
1. LINEアプリで公式アカウントを開く
2. 予約ボタン（リッチメニュー）をタップして予約画面が開くことを確認
3. テスト予約を1件入れてみる
4. スプレッドシートの「予約」シートにデータが入ることを確認
5. Googleカレンダーにイベントが作成されることを確認

## マニュアルの書き方の要件

- 対象読者：IT知識のない店舗オーナー（30〜50代）
- 文章：丁寧で分かりやすい日本語
- 構成：表紙・目次・各STEPの説明・よくある質問・困ったときの連絡先
- 各STEPには「スクリーンショット貼り付け箇所」として「【スクリーンショット：〇〇の画面】」と記載してください
- 注意事項・ポイントはコールアウトボックス（例：⚠️注意 / 💡ポイント）で強調してください
- 所要時間の目安をSTEP冒頭に記載してください（例：所要時間目安：30〜60分）
- 困ったときはサポートに連絡できることを末尾に記載してください
```

# プロンプト②：Config変更時マニュアル

```
あなたはSaaS製品の運用マニュアル作成の専門家です。
以下のシステム情報をもとに、IT知識のない飲食店・美容室・整体院などの店舗オーナーが
自分で設定変更できる「Config変更時マニュアル」を作成してください。

## システム概要

「LINE連動型予約システム」の設定はすべてWebブラウザから変更できます。
設定変更用画面のURLは店舗ごとに異なります（導入時に通知されたGAS WebアプリURLです）。

## 設定画面へのアクセス方法

1. PCのGoogleChromeブラウザを開く
2. 導入時に通知された「設定画面URL」を入力（GAS WebアプリURLそのもの）
3. Googleアカウントのログイン画面が出たら、導入時に使用したGoogleアカウントでログイン
4. 「予約システム設定画面」が表示される

## 変更できる設定一覧

### 基本設定
| 設定名 | 説明 | 変更するタイミング |
|---|---|---|
| 店舗名 | 予約画面に表示される店舗名 | 店舗名を変更したとき |
| 営業時間 | 予約受付の開始・終了時間 | 営業時間を変更したとき |
| 定休日 | 週の定休曜日（複数可） | 定休日を変更したとき |
| 予約枠の時間単位 | 15分・30分・60分など | 変更の機会は少ない |

### 予約設定
| 設定名 | 説明 | 変更するタイミング |
|---|---|---|
| 同時予約上限数 | 同じ時間に受けられる最大予約数 | スタッフ数が変わったとき |
| 先行予約可能日数 | 何日先まで予約できるか | 繁忙期・閑散期の調整 |
| 一括予約の最大件数 | 1回の操作で選択できる予約件数の上限 | 「1」に設定すると一括予約が無効になる |

### メニュー設定
- メニュー名と所要時間を追加・変更・削除できます
- JSON形式で入力します（例：[{"name":"カット","duration":60}]）
- ⚠️ メニューを削除すると既存予約のメニュー名は変わりません

### 担当者設定
- 担当者の追加・削除ができます
- 担当者機能を「OFF」にすると担当者選択なしで予約できます

### リマインダー設定
| モード | 説明 |
|---|---|
| ICS | 予約完了時にカレンダー追加ファイルをダウンロード（無料） |
| LINE | 予約の前日にLINEで通知（LINE Messaging API枠を消費） |

## 設定変更の手順

1. 変更したい項目の入力欄を修正する
2. 画面下部または各セクションの「保存」ボタンをクリック
3. 「設定を保存しました」と表示されれば完了
4. LIFFアプリ（予約画面）をリロードして変更が反映されていることを確認

## マニュアルの書き方の要件

- 対象読者：IT知識のない店舗オーナー（30〜50代）
- 文章：丁寧で分かりやすい日本語
- 各設定項目について「いつ変更するか」「変更するとどう変わるか」を分かりやすく説明してください
- 特に重要な注意事項（削除の注意など）は赤文字・ボックスで強調してください
- 設定変更後の確認方法を必ず記載してください
- 「よくある設定変更ケース」（例：お盆休みの設定方法、スタッフ追加の方法）をQ&A形式で記載してください
```

# プロンプト③：初期導入アカウント廃止時の引き継ぎマニュアル

```
あなたはSaaS製品の運用マニュアル作成の専門家です。
以下のシステム情報をもとに、「導入時に使用したGoogleアカウントを廃止・変更する際の
システム引き継ぎマニュアル」を作成してください。

## 背景・状況

このシステムはGoogleアカウントを使ってGoogleのクラウド上で動作しています。
導入時に使用したGoogleアカウント（以下「旧アカウント」）を廃止する場合、
事前に新しいGoogleアカウント（以下「新アカウント」）への引き継ぎが必要です。

⚠️ 旧アカウントを先に削除してしまうと、システムが完全に停止します。
必ずこのマニュアルの手順通りに作業してください。

## 引き継ぎが必要な理由

システムの動作は旧アカウントに紐づいています。具体的には：
- Googleスプレッドシート（データベース）が旧アカウントで作成されています
- GASスクリプト（バックエンドプログラム）が旧アカウントで動いています
- Googleカレンダーが旧アカウントで作成されています

## 引き継ぎ手順

### 事前準備（旧アカウントで作業）
1. 引き継ぎ先の新アカウントのGmailアドレスを用意する
2. 旧アカウントで以下のURLリストを手元に控える：
   - GAS WebアプリURL（設定画面・予約APIのURL）
   - スプレッドシートのURL
   - 予約カレンダーID（ConfigシートのreservationCalendarIdの値）
   - ブロックカレンダーID（Configシートのblock_input_calendar_idの値）

### STEP 1：スプレッドシートのオーナー権限を移譲
1. スプレッドシートを開く
2. 右上の「共有」ボタンをクリック
3. 新アカウントのGmailアドレスを入力して「編集者」権限で共有
4. 新アカウントにメールが届くので、承認してもらう
5. 再度「共有」ボタン → 新アカウント名の横の権限を「オーナー」に変更
6. 「オーナーを変更」をクリック

### STEP 2：Googleカレンダーの共有
1. Googleカレンダー（https://calendar.google.com）を旧アカウントで開く
2. 予約カレンダーとブロックカレンダーそれぞれを新アカウントと共有（「予定の変更および管理権限」で共有）

### STEP 3：新アカウントで新しいシステムを導入する
新しいアカウントでシステムを再導入します。
1. 新アカウントで導入サイト（https://reservation-onboarding.reservation-onboarding.workers.dev/）にアクセス
2. 「Googleアカウントで導入開始」から新アカウントでログイン
3. 新しいGAS WebアプリURLが発行される（必ず控える）

### STEP 4：旧スプレッドシートのデータを新システムにコピー
1. 旧スプレッドシートから以下のシートのデータをコピー：
   - 予約シート（過去の予約データ）
   - 顧客マスタシート（お客様情報）
   - Configシートの設定値（LINEトークン・メニュー・担当者情報など）
2. 新スプレッドシートに貼り付け

### STEP 5：LINE Developers の Webhook URL を更新
1. LINE Developersコンソールにログイン
2. STEP 3で発行した新しいGAS WebアプリURLをWebhook URLに設定

### STEP 6：動作確認
1. 新アカウントの設定画面URLで設定を確認
2. テスト予約を入れて動作を確認
3. 問題なければ旧アカウントの削除が可能

### ⚠️ 重要な注意事項
- 旧アカウントを削除する前に、必ず新システムの動作確認を完了してください
- LINE公式アカウントのWebhook URLを切り替えるまで、旧システムで予約を受け続けることができます
- 切り替え作業は予約の少ない時間帯（定休日の深夜など）に行うことを推奨します

## マニュアルの書き方の要件

- 対象読者：IT知識のない店舗オーナー（30〜50代）
- 文章：丁寧で分かりやすい日本語
- 手順はSTEP番号付きで明確に
- 各STEPの完了確認方法を記載
- 危険な操作（旧アカウントの削除）の前には強調した警告を入れる
- 作業の所要時間目安を記載（全体で2〜3時間程度）
- 不明点はサポートに連絡するよう促す
```

# プロンプト④：不具合発生時マニュアル

```
あなたはSaaS製品の運用マニュアル作成の専門家です。
以下のシステム情報をもとに、IT知識のない飲食店・美容室・整体院などの店舗オーナーが
不具合発生時に自分でできることを確認できる「不具合発生時マニュアル」を作成してください。

## システム概要

「LINE連動型予約システム」はGoogleクラウド上で動作しており、以下が主な構成です：
- LIFFアプリ（お客様の予約画面）：LINE内ブラウザで開くWebページ
- GASバックエンド：Google Apps Script（Googleサーバー上で動作するプログラム）
- スプレッドシート：予約データ・顧客データが保管されている
- Googleカレンダー：予約イベントが自動作成される
- LINE Messaging API：LINEメッセージの受信・送信

## よくある不具合と対処方法

### 不具合①：お客様が「予約できない」と言っている
確認手順：
1. スプレッドシートの「エラーログ」シート（非表示になっている場合あり）を確認する
   → 右下の「+」ボタンで隠れているシートを表示できます
   → エラーが記録されていれば内容を確認してください
2. 設定画面（GAS WebアプリURL）を開いて設定を確認する
   → 営業時間・定休日・メニューが正しく設定されているか確認
3. Configシートの「reservationCalendarId」に正しいカレンダーIDが設定されているか確認
4. Googleカレンダーにブロック予定が入りすぎていないか確認

### 不具合②：LINEで「予約」と送ったのに返信がこない
確認手順：
1. LINE Developersコンソール（https://developers.line.biz）にログイン
2. 「Messaging API設定」→「Webhook URL」が正しいGAS URLになっているか確認
3. 「Webhook URLを検証」ボタンをクリックして「成功」になるか確認
4. 失敗する場合はGAS URLが変わっている可能性があるのでサポートに連絡

### 不具合③：予約画面（LIFF）が「エラーが発生しました」と表示される
確認手順：
1. 予約画面のURL（liff.htmlのURL）を確認する
   → URLの末尾に「?gasApiUrl=https://...」と「&liffId=...」が含まれているか確認
   → 含まれていない場合はリッチメニューの設定を確認
2. GAS WebアプリURLが有効か確認する
   → ブラウザでGAS WebアプリURLを開いてみる
   → 設定画面が表示されればURLは有効
3. 解決しない場合はサポートに連絡

### 不具合④：Googleカレンダーに予約が入らない
確認手順：
1. スプレッドシートの「予約」シートに予約データは入っているか確認
   → データが入っていればカレンダーだけの問題
2. Configシートの「reservationCalendarId」の値を確認
   → Googleカレンダーのカレンダーの「設定と共有」→「カレンダーのID」と一致しているか確認
3. そのカレンダーにGASスクリプトからのアクセス権限があるか確認
   → カレンダーの共有設定で導入時のGoogleアカウントが「予定の変更および管理権限」を持っているか確認

### 不具合⑤：予約管理画面（kanri.html）にアクセスできない
確認手順：
1. URLの末尾に「?page=kanri」が含まれているか確認
2. 導入時に使用したGoogleアカウントでログインしているか確認
3. 別のGoogleアカウントでログインしている場合はブラウザのシークレットモードを使用

## エラーログの確認方法

1. スプレッドシートを開く
2. 画面下部のシートタブ一覧から「エラーログ」シートを探す（非表示の場合は右下の「+」を右クリック→「シートを表示」）
3. 「日時」「関数名」「エラー内容」の列を確認する
4. 最新のエラーを確認し、内容をサポートに連絡する際にコピーして添付する

## サポートへの連絡方法

以下の情報をまとめてサポートに連絡してください：
1. エラーが発生した日時
2. どの操作をしたときに発生したか
3. エラーログシートの内容（スクリーンショットまたはコピーペースト）
4. GAS WebアプリURL（設定画面のURL）

## マニュアルの書き方の要件

- 対象読者：IT知識のない店舗オーナー（30〜50代）
- 文章：丁寧で分かりやすい日本語
- 「症状→確認手順→対処法→それでも解決しない場合はサポートへ」の構成を徹底する
- 各不具合をフローチャートや判断ツリー形式で表現する（「はい/いいえ」で次のステップに進む）
- エラーログの見方を図解で説明する
- 「自分で解決できないと判断する基準」を明確に記載する
- 問い合わせ時に必要な情報のチェックリストを末尾に掲載する
```
