# LINE連動型予約システム テスト仕様書 v2.0
**作成日**: 2026-06-16（システム設計書 v2.0 に基づく）

---

## 1. テスト方針

### 1.1 テスト種別

| 種別 | 対象 | 実施方法 |
|---|---|---|
| **UT（単体テスト）** | GAS 関数 | GAS内 `runAllTests()` 関数 |
| **IT（結合テスト）** | LIFF ↔ GAS API | Playwright（GitHub Actions） |
| **ST（システムテスト）** | エンドツーエンド | 手動 + Playwright |
| **AT（受け入れテスト）** | ユーザー視点 | 手動確認 |

### 1.2 テスト環境

| 項目 | 内容 |
|---|---|
| テスト用スプレッドシート | 本番とは別のテスト用SS |
| テスト用GASデプロイ | 本番とは別のデプロイURL |
| テスト用LINEアカウント | テストユーザー複数アカウント |
| ブラウザ | Chrome（PC） / Safari（iOS LIFFアプリ） |

## 3. 自動化テスト設計

### 3.1 GAS 単体テスト（推奨構成）

GAS エディタから `runAllGASTests()` を実行することで全テストを一括チェックする。

```javascript
// GAS テストランナー（実装予定）
function runAllGASTests() {
  const results = [];
  results.push(...testGetInitData());
  results.push(...testRegisterCustomer());
  results.push(...testCreateBooking());
  results.push(...testCreateBulkBookings());
  results.push(...testGetAvailableSlots());
  results.push(...testArchive());
  
  const passed = results.filter(r => r.passed).length;
  const failed = results.filter(r => !r.passed);
  Logger.log(`テスト結果: ${passed}/${results.length} 件合格`);
  if (failed.length > 0) {
    Logger.log('失敗したテスト:');
    failed.forEach(r => Logger.log(`  ✗ ${r.name}: ${r.error}`));
  }
}
```
