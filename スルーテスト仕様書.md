## テスト前準備：デプロイ手順（コード変更があった場合）

> **コードを変更していない場合はこのセクションをスキップして「テスト環境」へ進んでください。**

スルーテスト実施前に、最新コードのデプロイと自動テスト合格を確認します。

### 自動デプロイ & 自動テスト実行（ワンコマンド）

PowerShellを開き、以下を実行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
cd C:\Users\user\reservation-liff-app
.\deploy-and-test.ps1
```

#### 実行内容（スクリプトが自動で行う）

| ステップ | 内容 |
|---|---|
| STEP 1 | `clasp push` でGASにコードをプッシュ |
| STEP 2 | `clasp deploy` でデプロイを最新バージョンに更新（URLは変わらない） |
| STEP 3 | GAS `runTests` APIを呼び出してユニットテスト22件を実行 |
| STEP 4 | 結果をコンソールに出力。証跡はスプレッドシート「テスト結果」シートに自動保存 |

#### 合格確認

```
  Total    : 22
  Passed   : 22
  Failed   : 0
  All tests passed! Ready for through-test.
```

上記が表示されたらスルーテストに進んでください。

> **Failed が 1件以上の場合**: コードを修正後、再度スクリプトを実行してください。  
> スルーテストは **全自動テスト合格後** に実施します。

> **本番店舗環境への反映（参考）**: LIFF は `git push` → GitHub Pages。GAS は Cloudflare `/admin` の「コード配信」を使用。開発用の `clasp push` は上記スクリプト経由。
